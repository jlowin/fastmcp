Transforms modify components as they flow through your server. They're the mechanism behind `mount(namespace="...")` and can be used directly for custom transformations.

## Why Transform

When composing servers, you may encounter naming conflicts:

```python
weather = FastMCP("Weather")
calendar = FastMCP("Calendar")

@weather.tool
def get_data() -> str:
    return "Weather"

@calendar.tool
def get_data() -> str:  # Same name!
    return "Calendar"

main = FastMCP("Main")
main.mount(weather)
main.mount(calendar)  # Conflict: which get_data?
```

Transformations solve this by prefixing or renaming components.

## Namespacing

The most common transformation is namespacing, which prefixes all component names:

```python
main = FastMCP("Main")
main.mount(weather, namespace="weather")
main.mount(calendar, namespace="calendar")

# Tools are now:
# - weather_get_data
# - calendar_get_data
```

### Namespace Rules

| Component Type | Original | With `namespace="api"` |
|----------------|----------|------------------------|
| Tool | `my_tool` | `api_my_tool` |
| Prompt | `my_prompt` | `api_my_prompt` |
| Resource | `data://info` | `data://api/info` |
| Template | `data://{id}` | `data://api/{id}` |

Tools and prompts get an underscore prefix. Resources get a path-style prefix.

## Tool Renaming

For more control, rename specific tools with `tool_names`:

```python
main.mount(
    weather_server,
    namespace="weather",
    tool_names={"verbose_tool_name": "short"}
)

# "verbose_tool_name" → "short" (explicit rename, bypasses namespace)
# "other_tool" → "weather_other_tool" (namespace applied)
```

Explicit renames take precedence over namespacing.

## Transform API

Under the hood, namespacing and tool renaming use **transforms**. You can use transforms directly for more control.

### Available Transforms

| Transform | Purpose |
|-----------|---------|
| `Namespace` | Adds prefixes to names and path segments to URIs |
| `ToolTransform` | Modifies tool schemas (rename, description, tags) |

### Server-Level Transforms

Add transforms to your server with `add_transform()`. Server transforms apply to all components from all providers:

```python
from fastmcp import FastMCP
from fastmcp.server.transforms import ToolTransform, Namespace
from fastmcp.tools.tool_transform import ToolTransformConfig

mcp = FastMCP("Server")

@mcp.tool
def my_tool() -> str:
    return "hello"

# Rename a tool
mcp.add_transform(ToolTransform({
    "my_tool": ToolTransformConfig(name="better_name")
}))

# Add a namespace to everything
mcp.add_transform(Namespace("api"))
```

### Provider-Level Transforms

You can also add transforms to individual providers:

```python
from fastmcp.server.providers import FastMCPProvider
from fastmcp.server.transforms import Namespace

sub_server = FastMCP("Sub")

@sub_server.tool
def greet() -> str:
    return "hello"

# Create provider with namespace transform
provider = FastMCPProvider(sub_server)
provider.add_transform(Namespace("api"))

# Register with main server
main = FastMCP("Main", providers=[provider])
# Tool is now: api_greet
```

### Transform Stacking

Transforms stack - first added is innermost:

```python
from fastmcp.server.transforms import Namespace, ToolTransform
from fastmcp.tools.tool_transform import ToolTransformConfig

provider = FastMCPProvider(server)
provider.add_transform(Namespace("api"))  # Applied first
provider.add_transform(ToolTransform({   # Sees namespaced names
    "api_verbose": ToolTransformConfig(name="short")
}))

# "verbose" → "api_verbose" (namespace) → "short" (rename)
```

## ToolTransformConfig Options

When using `ToolTransform`, each tool can be configured with:

```python
from fastmcp.tools.tool_transform import ToolTransformConfig

config = ToolTransformConfig(
    name="new_name",           # Rename the tool
    description="New desc",    # Change description
    tags={"admin"},           # Add tags
    # ... and more
)
```

See [Tool Transformation](/patterns/tool-transformation) for argument-level transforms using `Tool.from_tool()`.

## Use Cases

### Avoiding Conflicts

```python
# Two servers with same tool names
main.mount(server_a, namespace="a")
main.mount(server_b, namespace="b")
```

### API Versioning

```python
# Mount different versions
main.mount(v1_server, namespace="v1")
main.mount(v2_server, namespace="v2")
```

### Aliasing Long Names

```python
mcp.add_transform(ToolTransform({
    "get_user_authentication_status": ToolTransformConfig(name="auth_status"),
    "retrieve_configuration_settings": ToolTransformConfig(name="get_config"),
}))
```
